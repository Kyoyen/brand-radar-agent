场景挖掘

为什么做这个？

在快消品牌营销（KFC）的实际工作中，竞品社媒监控是高频、重复、规则清晰的典型 Agent 化场景。

⏰

时间成本高

每周花 3-4 小时人工翻阅小红书/抖音/微博，截图，整理笔记

📋

格式不统一

每个人整理方式不同，结果不可对比，无法追踪竞品趋势变化

🐌

传递链路长

分析完需要手动发给团队，时效性差，往往下午报告才到早上的热点

🎯

洞察不够深

人工整理停留在「描述」层面，缺少从我方品牌视角的针对性判断

产品迭代

三轮迭代过程

不一步到位，每轮针对真实问题进行小步验证，逐步扩展能力边界。

V1
手动 Prompt 版
验证 POC

✅ 做了什么

  • 单一通用 Prompt 替代人工整理
  • 人工粘贴内容 → GPT 分析 → 输出文字
  • 验证 AI 分析竞品内容的可行性

❌ 发现问题

  • 输出格式每次不一致
  • 缺少品牌视角，结论太泛
  • 数据采集仍靠手工，瓶颈未解
验证结论：AI 分析有价值，但不结构化就无法规模化 → 必须上 Schema 约束
V2
结构化输出版
Structured Output

✅ 改进内容

  • Pydantic Schema 约束 JSON 输出
  • 加入「我方品牌」上下文参数
  • 支持多竞品批量处理 + 威胁排序

📈 效果提升

  • 格式一致率 100%（可程序处理）
  • 准确率从约 60% → 约 80%
  • 3 个竞品批量分析仅需 ~30s
仍存在问题：数据采集仍手动（约 20 分钟/次），无推送机制 → 需要工具调用
V3
完整 Agent 版
Function Calling

✅ 核心升级

  • 4 个工具：搜索/热点/飞书/存档
  • LLM 自主决定调用时机和顺序
  • 飞书 Webhook 自动推送团队
  • 历史存档 + 趋势对比能力

📈 最终效果

  • 端到端耗时：~2 分钟
  • 人工介入降至「核验」环节
  • Prompt 准确率 ≥ 85%（3轮迭代）
  • 团队响应速度提升 ~50%
最终成果：从 3-4 小时/周人工整理 → 约 2 分钟自动完成，效率提升 ~90%

Prompt 迭代对比

三轮 Prompt 演进，展示从无约束到精准控制的优化过程。

# V1：通用 Prompt，输出自由文本（问题：格式随机） SYSTEM_PROMPT_V1 = """你是一位资深品牌营销分析师。 请帮我分析以下竞品的社媒内容，重点关注： 内容主题、传播策略、用户互动方式、以及可借鉴的打法。""" # ❌ 问题：没有格式约束，每次输出结构完全不同 # ❌ 问题：没有「我方品牌」视角，分析太泛 # ❌ 问题：无法程序化处理输出结果
# V2：Pydantic Schema + 品牌上下文（准确率 60% → 80%） class CompetitorBrief(BaseModel): competitor_name: str overall_strategy: str # 整体策略一句话总结 threat_level: ThreatLevel # 强制枚举：高威胁/中等/低/可借鉴 core_messaging: list[str] # 核心传播关键词 actionable_recommendations: list[str] # 可执行建议 # V2 关键升级：response_format 强制结构化输出 response = client.beta.chat.completions.parse( model="gpt-4o-mini", messages=messages, response_format=CompetitorBrief, # ← 核心改动 temperature=0.3, # ← 降低随机性 ) # ✅ 效果：格式 100% 一致，可直接程序化处理 # ❌ 仍存在：数据采集还是手动
# V3：Function Calling，LLM 自主决策工具调用（完整 Agent） # 工具注册：定义 4 个工具，Agent 自主决定何时调用 TOOLS = [ {"type": "function", "function": { "name": "search_social_content", "parameters": { "brand": {"type": "string"}, "platform": {"type": "string", "enum": ["小红书", "抖音", "微博"]}, }, }}, # ... send_feishu_report, get_trending_topics, save_to_history ] # Agent 主循环（ReAct 模式） while turn < MAX_TURNS: response = client.chat.completions.create( model="gpt-4o-mini", messages=messages, tools=TOOLS, tool_choice="auto", # ← LLM 自主决定是否调用工具 ) if finish_reason == "stop": break # 任务完成 for tool_call in msg.tool_calls: result = execute_tool(tool_call.function.name, args) messages.append({"role": "tool", "content": result}) # ✅ 效果：全自动，~2 分钟完成，飞书自动推送
