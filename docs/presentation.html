工作原理

输入一句话，Agent 完成剩下所有

基于 ReAct 模式——每一步先推理，再行动，再观察，循环直到任务完成。

1
自然语言输入
无需学习命令，描述你要做的事
2
场景自动识别
关键词匹配 + LLM 兜底，找到对应场景
3
ReAct 循环执行
Think → 调用工具 → Observe → 继续
4
结果推送
飞书卡片 / 邮件，格式统一可复用
5
经验归档
执行过程提炼存档，下次自动召回
三层记忆系统

越用越聪明，不只是执行工具

Agent 不会每次从零开始。执行中压缩上下文，任务后保存历史，经验自动沉淀供未来召回。

L1
执行中压缩
每 N 次工具调用自动生成进度快照，防止超出模型上下文限制，长任务稳定运行。
L2
任务后持久化
完成后自动存档执行摘要、关键发现、下次待跟进事项，下次同类任务自动加载。
L3
经验归档召回
将执行过程提炼为结构化经验：解决思路、工具顺序、踩过的坑。相似任务自动引用。
迭代路径

四轮迭代，从验证到框架

每一轮解决上一轮留下的核心问题，完整展示从「能跑通」到「可扩展」的工程演进。

V1 · v1-poc 分支
验证可行性
单文件脚本，硬编码数据，证明 AI 可以完成竞品监控这件事。留下的问题：输出不稳定，结果难以解析。
单脚本硬编码数据OpenAI Chat
V2 · v2-structured 分支
结构化输出
引入 Pydantic Schema 约束输出格式，结果变为可程序化处理的 JSON，威胁等级分级。留下的问题：Agent 只能直线执行，无法自主决策工具调用顺序。
Pydantic v2结构化 JSON威胁分级
V3 · v3-agent 分支
工具调用 + ReAct 循环
接入 OpenAI Function Calling，Agent 自主决策调用哪些工具、调用顺序，结果推送飞书卡片。留下的问题：场景单一，扩展需要改代码，上下文没有记忆。

3 步运行第一个任务

不需要学习新的命令语法——用自然语言描述你要做的事，Agent 自动处理剩下的一切。

terminal — brand-radar-agent
# 1. 安装依赖
$ pip install -r requirements.txt

# 2. 配置 API Key（.env 文件填写一行）
$ cp .env.example .env
# LLM_PROVIDER=deepseek 或 openai / anthropic

# 3. 运行（自然语言描述任务）
$ python run.py "帮我分析麦当劳今天在小红书发了什么"
